# STYLE MENU (for picking, 2026-07-09)

How to use this file: pick style numbers, say "demo 4, 11, 19 on a forest
tile" (or a menu, a survivor, anything). A demo = 1-3 samples, judged,
then locked or dropped. Stars (*) = my picks for THIS game.

## How to get the best out of me (short version)

1. REFERENCE FIRST. A screenshot or a game name beats ten adjectives.
   assetFace.png produced the best result of this project so far in one
   round. "Like Minit" tells me more than a paragraph.
2. SMALL PROOF, THEN BATCH. Ask for 1-3 samples, inspect, then I mass-
   produce in the approved style. Never order 10 unproven things.
3. KNOW WHAT I AM. I draw by writing rules, not by hand. Anything
   expressible as rules (grids, geometry, dither, silhouettes, patterns,
   type) I do very well. Anything needing freehand organic judgment
   (faces from scratch, flowing hair, painterly strokes) I do poorly.
   The tiers below are exactly that line.
4. LOCK STYLES AS LAW. When a demo passes, its rules get written down
   and the generator code kept. Every later asset in that style is then
   cheap and perfectly consistent. The avatar pipeline works this way.
5. ITCH.IO SPLIT. Buy organic/detailed things (big illustrations,
   character sheets, Steam capsule art) from itch. I do everything that
   needs consistency and code integration (UI, icons, tiles, markers,
   motion). I can also RESTYLE itch assets: quantize to our palette and
   pixel grid so bought art looks native. Keep license notes per pack.

## TIER S - bet on these (proven or rule-pure)

1.  * CHUNKY 1-BIT PIXEL. White on black, fat pixels, hard edges.
    Sprites, icons, tiles. The house default. (google: Minit, World of
    Horror)
2.  * DITHER SHADING 1-BIT. Checker/noise dithering for gradients, fog,
    smoke. Pairs with 1 for fog of war. (google: Return of the Obra
    Dinn, old Macintosh art)
3.  * SILHOUETTE. Solid black or white shapes, zero interior detail.
    Perfect for crowds, trees, skylines. Menace without gore.
4.  * GEOMETRIC MINIMAL. Circles, squares, lines only, nothing
    representational. Markers, maps, abstract UI. (google: Mini Metro)
5.  * BITMAP-TYPE UI. Tiny pixel fonts as the main visual element.
    Menus, HUDs, tooltips. The current main menu is this.
6.  OUTLINE / HOLLOW PIXEL. 1px white outlines, unfilled. Ghost states,
    locked tiles, blueprints, fog-hidden objects.
7.  ICON-EMBLEM TILES. One centered symbol per tile or card (the big
    tree, tile C). Extremely legible at small sizes.
8.  PATTERN FILL. Repeating micro-marks as area texture (the thicket,
    tile B). Terrain zones, danger areas.
9.  MICRO SPRITES. 8x8 to 16x16 objects and creatures. The resource
    icons lab proved this size.
10. PIXEL SIGILS. Abstract personal marks/glyphs. Survivor signatures,
    faction stamps, save slots.

## TIER A - strong, small constraints

11. * ONE ACCENT COLOR. Everything B&W plus a single color used rarely
    (danger red, signal amber). Huge payoff for alarms/hordes/blood-
    free violence. (google: Downwell)
12. GRAYSCALE PIXEL. 3-5 grays instead of pure 1-bit. Your avatars
    already live here (navy/gray/white). Softer, still minimal.
13. TERMINAL / PHOSPHOR. Monochrome CRT computer UI, scanline vibe.
    Natural for the bunker systems and the rewind device. (google:
    fallout terminal, Duskers)
14. ASCII-FLAVORED. Letters and symbols as world elements. Could style
    the whole map or just accents. I am, literally, best at text.
15. PICTOGRAM LANGUAGE. Consistent little symbol set that teaches
    mechanics without words (fits the no-tutorial-text law).
16. FLAT VECTOR MINIMAL. Crisp flat shapes, not pixelated. Clean modern
    panels if we ever want contrast with the pixel world.
17. BLUEPRINT / SCHEMATIC. Thin lines, dashes, right angles on dark.
    Build mode, bunker floor plans. (google: blueprint UI game)
18. STENCIL / SIGNAGE. Blocky spray-stencil lettering and marks.
    Survivor-made signs, warnings. (google: stencil military signage)
19. NODE GRAPH. Dots connected by lines. Skill trees, survivor bonds,
    prestige map. (google: constellation skill tree)
20. DOT MATRIX / LED. Everything built from round dots. Counters,
    tickers, day display. (google: flipdot display)

## TIER B - workable with care

21. MICRO ISOMETRIC. Tiny 2:1 isometric tiles. Pretty but doubles asset
    cost; fine for boxes/buildings, risky for organic shapes.
22. 4-COLOR GAMEBOY PALETTE. Mood variant of grayscale. (google: Game
    Boy palette pixel art)
23. YOUR AVATAR STYLE EXTENSIONS. New people/props built ON your exact
    base pixels. Proven, but only works as edits of your art, never
    from scratch.
24. PROCEDURAL GRAIN / STATIC. Noise, fog, film grain as atmosphere
    layers. Trivial for me (it IS noise), classy when subtle.
25. PAPERCUT LAYERS. Flat stacked shades for depth/parallax
    backgrounds. Simple shapes only.

## TIER C - I will do these badly; use itch.io instead

26. DETAILED ORGANIC PIXEL ART. Realistic people, animals, flowing
    hair. This session already proved it.
27. PAINTERLY / ILLUSTRATED. Any brush feel. Not my medium.
28. ANIME / CARTOON CHARACTERS. No.
29. LARGE KEY ART (Steam capsule, posters). Outsource; I can mock
    layouts but not finish them.
30. HAND-DRAWN WOBBLY LINEWORK. Fakeable procedurally but reads janky.

## My recommended direction for this game

1 + 2 + 3 + 5 as the core (chunky 1-bit world, dithered fog, silhouette
hordes, bitmap-type UI), 11 as the only color in the game (one red,
reserved for real danger), 13 for the bunker/rewind screens, 19 for the
prestige layer. Your circle avatars stay as the one soft element.
Coherent, fully in my wheelhouse, and everything stays consistent
because each style is a locked rule set.
